# AI JSON 输出通用规范

当本次任务要求你输出 JSON 时，你必须严格遵守以下规则（适用于所有 JSON 输出任务，与具体业务无关）：

1. **纯 JSON**：回复正文必须且仅能是一个 JSON 值（对象 `{...}` 或数组 `[...]`），可被标准 JSON 解析器直接解析。
2. **首尾干净**：第一个非空白字符必须是 `{` 或 `[`；最后一个非空白字符必须是对应的 `}` 或 `]`。禁止在 JSON 前后添加任何文字、标题、说明或礼貌用语。
3. **禁止 Markdown 围栏**：不要使用 ```json ... ``` 或其它代码块包裹。
4. **语法合法**：
   - 所有键名与字符串值使用双引号 `"`，禁止单引号
   - 禁止 trailing comma（如 `{"a":1,}` 或 `[1,2,]`）
   - 禁止 JSON 内注释（`//`、`/* */`）
   - 字符串内的换行、引号、反斜杠须正确转义（`\n`、`\"`、`\\`）
5. **单一根节点**：只输出一个 JSON 根，不要输出多个并列 JSON 块。
6. **中文内容**：描述性中文应写在 JSON 字符串值内部，不要写在 JSON 结构外部。

若任务说明与以上规则在**数据结构**上冲突，以任务说明为准；但**输出洁净度与语法合法性**仍必须遵守本节。
