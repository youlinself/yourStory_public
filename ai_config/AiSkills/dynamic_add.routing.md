## 技能 `dynamic_add` — AI 侧协议（触发词）

程序已注册本技能。需要**新增未定义**的运行时数据时，在叙事回复中嵌入标记（**不要**输出 JSON）：

```
[[DYN_ADD:分类|来源与要点说明]]
```

### 分类一览

| 标记中的分类 | schema_id | 含义 |
|-------------|-----------|------|
| 物品、道具 | loot_item | 消耗品、材料、货币、垃圾 |
| 装备、武器 | loot_weapon | 武器、护甲、工具 |
| 技能 | runtime_skill | 新技能 |
| NPC、角色 | runtime_npc | 新 NPC |
| 子地点、地点 | runtime_key_node | 小区、楼栋、店铺等 |
| 区域、城区 | runtime_region | 新城区/片区 |

也可直接写 schema_id，如 `[[DYN_ADD:loot_item|...]]`。

### 示例

- `[[DYN_ADD:物品|生锈的医疗柜，应有消毒用品]]`
- `[[DYN_ADD:装备|辐射巢穴里的锈蚀货架]]`
- `[[DYN_ADD:技能|老猎人传授的包扎术]]`
- `[[DYN_ADD:NPC|断桥集市的流浪商人]]`
- `[[DYN_ADD:子地点|南郊康宁小区7栋501，住宅]]`
- `[[DYN_ADD:区域|南郊，城郊住宅带]]`

**叙事与数据同轮**：首次写出可对话 NPC 或可复现地名时，必须与上述标记成对出现，禁止只写正文。

**物品/装备设定一致性**：生成消耗品或装备时，须结合当前 `novel_type` 与世界设定、`|` 后来源说明，不得偏离题材、时代与科技水平。

### 单次多条与上限

- 同一轮回复可写**多条**标记（如一次搜刮多件），每条独立一行。
- 程序默认**最多处理 5 条**（`dynamic_add_registry.json` → `limits.max_per_response`）；超出标记会替换为「已达上限」提示。
- 2 条及以上时合并为**一次**批量生成请求（减少 API 调用）。

### 程序行为（AI 无需执行）

1. 解析全部标记 → 截断至上限 → 选定各条 schema  
2. 单条：一次生成；多条：批量生成 prompt，**一次**调用 AI 得到 `entries`  
3. 逐条查重后写入对应总表  
4. 将各标记替换为【物品名】等，并可继续叙事  

完整实现见 `src/ai_skills/dynamic_add_service.gd`。
